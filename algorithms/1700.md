## 핵심 아이디어
- 학생 queue를 직접 시뮬레이션할 필요 없음
	- queue 시뮬레이션을 counting으로 단순화하는 패턴
- 필요한 것은 "순서"가 아니라 "종류별 남은 학생 수"

## base case
- 현재 sandwich를 원하는 학생 수가 0인 경우
  -> 더 이상 진행 불가 → 남은 학생 수 반환

- 모든 sandwich를 순회한 경우
  -> 모든 학생이 식사 완료 → 0 반환

## 자료구조 선택
- Dictionary (Counter)
  - [샌드위치 종류: 해당 샌드위치를 원하는 학생 수]

```swift
var counter = Dictionary(grouping: students, by: { $0 })
    .mapValues { $0.count }
```

## 왜 students를 기준으로 grouping하는가?
- sandwich는 순서대로 하나씩 처리됨 + students는 queue회전으로 순서 추적이 번거로움
- 실제 필요한 정보: "현재 sandwich"를 원하는 학생이 존재하는가?
- 따라서 students를 기준으로 count를 만들면 간단하게 basecase 판단 가능

## 동작 흐름
- students 기준 counter 생성
- sandwiches 순회
- 각 sandwich마다
	- 해당 sandwich를 원하는 학생이 없으면 종료
		- return 못 먹은 학생들 (= 반대 sandwich를 원하는 학생 수 = sandwich ^ 1)
			- sandwich는 0과 1뿐. 1과 XOR하면 무조건 토글된 값이 나옴
	- 있으면 counter 감소
