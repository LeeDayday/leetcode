## Heap 자료구조
- Heap의 특성
	- minHeap의 경우, 부모 <= 자식이 보장됨
	- maxHeap의 경우, 부모 >= 자식이 보장됨
	- 일반적인 heap 구현은 완전 트리 구조를 사용한다.
	- 완전 이진 트리의 경우, 배열로 효율적인 구현이 가능하다.
	  (부모: i, `왼쪽 자식: 2 * i`, `오른쪽 자식: 2 * i + 1` (단, 1-based index 기준))

- Heap의 특성을 유지하는 방법
	- insert
		- 가장 마지막 leaf node 에 추가 (append)
		- 해당 index와 부모 index를 비교하며 minHeap/maxHeap 특성을 만족할 때까지 swap
		- (해당 index: i 라고 하면, 부모 index: i/2)
		- 해당 index가 1보다 크고 heap 특성을 만족할 때까지 swap 반복
		  (예: minHeap: while i > 1 && array[i] < array[i / 2])
		- O(logN) (완전 이진 트리의 높이)
	- pop
		- 마지막 노드를 root(가장 첫번째 node. 0 부터 시작했다면 [0], 1부터 시작했다면 [1])로 올린뒤
		- 아래로 내려가며 heapify 수행
		- O(logN)
	- heapify
		- 특정 Index에서 마지막 index까지 밑으로 swap 진행 (heap 특성을 만족할 때까지)
		- swap을 진행할 땐, 자식 노드 중 우선순위가 더 높은 노드와 swap 
		  (minHeap이라면 더 작은 자식)
		- heapify는 특정 node에서 heap 조건을 맞추는 과정 (sift-down)을 의미
		- O(logN)
- build heap: 배열 전체를 heap으로 만드는 과정
- sift-down: 특정 노드를 아래로 내려가며 heap 조건을 맞추는 과정
- sift-up: 삽입 시 위로 올라가며 heap 조건을 맞추는 과정
