## 핵심 아이디어
- 파스칼의 삼각형 생성
- 현재 행의 값은 이전 행의 값을 이용해 계산
- 가장자리는 항상 1
- 내부 값은 바로 위 두 수의 합

## 패턴
- DP (Bottom-Up)
- 2차원 배열 생성
- 이전 상태를 이용하여 현재 상태 계산

## 규칙

```text
        1
      1   1
    1   2   1
  1   3   3   1
1   4   6   4   1
```

- 양 끝은 항상 1

```text
row[0] = 1
row[last] = 1
```

- 내부 값

```text
row[j] = prevRow[j - 1] + prevRow[j]
```

## 풀이 흐름
- 첫 번째 행부터 생성
- 현재 행의 크기는 rowIndex + 1
- 양 끝을 1로 설정
- 내부 값은 이전 행을 참고하여 계산
- 결과 배열에 추가

## DP 정의

```swift
result[i]
```

- i 번째 행

## 점화식

```swift
result[i][j]
=
result[i - 1][j - 1]
+
result[i - 1][j]
```

## Base Case

```swift
[1]
```

- 첫 번째 행

## 구현 패턴

```swift
for i in 0..<numRows {
    var row = Array(repeating: 1, count: i + 1)

    for j in 1..<i {
        row[j] =
            result[i - 1][j - 1]
            + result[i - 1][j]
    }

    result.append(row)
}
```

## 시간복잡도

- 총 생성되는 원소 개수

```text
1 + 2 + 3 + ... + n
```

```text
O(n²)
```

## 공간복잡도

- 결과 저장

```text
O(n²)
```

## 이 문제에서 중요한 사고

- 현재 값을 직접 계산하는 것이 아니라
- 이전 행의 정보를 이용

```text
현재 상태 = 이전 상태들의 조합
```

- DP 의 가장 기본적인 형태

## 자주 연결되는 패턴

- Triangle DP
- Combinatorics

## 기억할 포인트

- 양 끝은 항상 1
- 내부 값은 위의 두 수의 합
- "현재 행은 이전 행으로부터 생성된다"
- Bottom-Up DP 의 대표 입문 문제
