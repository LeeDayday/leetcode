
## 핵심 아이디어
- 각 word를 숫자(weight)로 변환
- 알파벳을 weights 배열의 index와 매핑
- 계산된 weight를 modulo 26 수행
- 결과를 역순 알파벳(reverse alphabet)으로 변환

## 패턴
- Character → Index 변환
- Lookup Table 활용
- 누적 계산(Aggregation)
- Modulo Mapping
- ASCII 연산

## 풀이 흐름
- word 순회
- 각 문자의 알파벳 index 계산
- weights[index] 누적
- 최종 weight % 26 계산
- reverse alphabet 규칙으로 문자 변환
- 결과 문자열에 추가

## 알파벳 → Index 변환

```swift
let idx = Int(ch.asciiValue!) - Int(Character("a").asciiValue!)
```

예시

```text
a → 0
b → 1
c → 2
...
z → 25
```

## Lookup Table 패턴

```swift
weight += weights[idx]
```

- 문자 자체를 사용하지 않고
- index를 통해 미리 준비된 값을 조회

## Modulo Mapping

```swift
weight % 26
```

- 결과를 알파벳 범위(0~25)로 압축

## Reverse Alphabet Mapping

문제 조건

```text
0  → z
1  → y
2  → x
...
25 → a
```

공식

```swift
Character(
    UnicodeScalar(
        Int(Character("a").asciiValue!)
        + 25
        - result
    )!
)
```

예시

```text
result = 0  → z
result = 1  → y
result = 2  → x
result = 25 → a
```

## 시간복잡도
- 전체 문자 개수를 N이라 할 때
- O(N)

## 공간복잡도
- 결과 문자열 제외
- O(1)

## 이 문제에서 중요한 사고
- 문자를 숫자로 변환
- 숫자를 다시 문자로 변환
- "Lookup → Aggregate → Modulo → Mapping" 흐름

## 자주 연결되는 패턴
- ASCII 연산
- Character Mapping
- Hashing 느낌의 Encoding
- Lookup Table
- Modulo Arithmetic

## 기억할 포인트
- 알파벳 index는 ascii 차이로 계산 가능
- Lookup Table 문제는 배열 index 접근을 떠올리기
- Modulo는 범위를 압축할 때 자주 사용
- 문자 ↔ 숫자 변환은 ASCII 기반으로 처리 가능
- Reverse Mapping은 기준 문자('a', 'z')를 이용해 공식화 가능
