## 핵심 아이디어
- 각 가격에 대해 오른쪽에서 처음 만나는 `현재 가격 이하` 값을 할인
- 단순히 차례대로 뒤를 모두 확인하면 O(n^2)
- 더 효율적으로 찾기 위해 Monotonic Stack 사용

## 패턴
- Monotonic Stack
- Next Smaller or Equal Element
- 오른쪽에서 조건을 만족하는 첫 번째 값 찾기
- 할인 / 가격 보정 문제

## 풀이 흐름
- prices 를 왼쪽부터 순회
- stack 에는 아직 할인을 찾지 못한 index 저장
- 현재 가격이 stack top 가격보다 작거나 같다면 할인 가능
- 해당 index 의 가격에서 현재 가격을 빼기
- 현재 index 를 stack 에 추가

## Stack 에 저장하는 값
- 가격 자체가 아니라 index 저장
- index 를 저장해야 원본 배열의 값을 수정할 수 있음

## 할인 조건

```swift
prices[j] <= prices[i]
```

- i 번째 상품의 오른쪽에서
- 처음 만나는 i 가격 이하의 상품 가격을 할인

## Monotonic Stack 조건

```swift
while !stack.isEmpty && prices[stack.last!] >= prices[i]
```

- stack top 상품은 현재 가격으로 할인받을 수 있음
- 할인 처리 후 stack 에서 제거

## 시간복잡도
- 각 index 는 stack 에 한 번 들어가고 한 번 나옴
- O(n)

## 공간복잡도
- stack 사용
- O(n)

## 이 문제에서 중요한 사고
- “오른쪽에서 처음 만나는 조건 만족 값” → Stack
- 모든 쌍을 비교하면 O(n^2)
- 이전 값들이 현재 값으로 해결될 수 있는지 확인하는 방식

## 자주 연결되는 패턴
- Next Greater Element
- Next Smaller Element
- Daily Temperatures
- Stock Span
- Monotonic Stack

## 기억할 포인트
- 오른쪽의 첫 조건 만족 값 찾기 → Monotonic Stack
- 할인 문제지만 본질은 Next Smaller or Equal Element
- stack 에는 아직 답을 못 찾은 index 를 저장
