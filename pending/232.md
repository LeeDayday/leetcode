## 핵심 아이디어
- 스택 2개를 이용해 FIFO 구조 만들기
- 핵심은 "순서를 한 번 뒤집어서 FIFO를 만든다는 것"

## 스택의 역할
- stack1: push 전용 (입력 저장)
- stack2: pop/peek 전용 (출력 담당)

## 원리
- stack1 -> stack2로 옮기면서 순서가 뒤집힌다
- 이 상태에서 pop하면 FIFO가 된다
- 예:
	- stack1: [1, 2, 3]
	- -> stack2: [3, 2, 1]
	- -> pop: 1, 2, 3

## 동작
- push: stack1에 append
- pop
	- stack2가 비어 있으면
		- stack1 -> stack2로 모두 이동
	- stack2에서 pop
- peek
	- stack2가 비어 있으면
		- stack1 -> stack2로 모두 이동
	- stack2에서 last
- empty
	- stack1, stack2가 모두 비어 있는 경우
- stack2가 비어있는지 검사하는 이유
	- stack2에는 이미 "queue의 앞쪽 요소" 들이 들어 있을 수도 있기 때문
