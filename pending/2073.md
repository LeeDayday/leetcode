## 핵심 아이디어
- queue 문제라고 해서 풀이에서도 실제로 removeFirst(popFirst) / append 를 수행할 필요는 없다
- 핵심은 queue의 순환을 직접 시뮬레이션하는 것이 아니라,
  각 사람이 k번째 사람이 끝날 때까지 몇 번 티켓을 사는지 계산하는 것이다
- 삭제, 추가의 위치가 고정되어 있으니, 규칙을 통해 풀이가 가능하다.

## base case
- k번째 사람이 마지막 티켓을 사는 순간 종료

## 시뮬레이션 vs 규칙 찾기
- 실제 시뮬레이션(removeFirst, append 수행)으로 문제를 푼다면 
	- queue 회전으로 인해 k번째 사람의 위치를 계속 추적해야 함
	- 하지만 실제로 필요한 것은 각 사람이 몇 번 처리되는지
	- 따라서 사람마다 티켓 구매 횟수를 계산하여 합산하여 필요한 값을 구할 수 있다
- 규칙 찾기로 한다면
	- 무엇을 기준으로 동작이 쪼개지는 지를 세워야 함
		- k번째 사람 앞과 뒤로 다른 논리로 접근해야 함
		- 앞 사람: 앞 사람은 무조건 동작이 수행됨. 즉, min(앞사람 티켓 수, k번째 사람 티켓수) 만큼은 무조건 수행이 되어야 함
		- 뒷 사람: k번째에서 끝나기 직전 turn까지는 동작이 수행됨. 즉, min(k번째 티켓수 - 1, 뒷사람 티켓수) 만큼은 무조건 수행이 됨

## 동작 수행
- tickets을 순회하면서
	- 각 사람의 티켓 구매 횟수를 더한다
	- 앞 사람이라면 min(i번째 사람 티켓 수, k번째 사람 티켓 수)
	- 뒷 사람이라면 min(k번째 사람 티켓 수 - 1, i번째 사람 티켓 수)
